# Offline Maps

AlpiMaps was built from the ground up for offline use. When offline data packs are present the app automatically detects and loads them, enabling a full set of features without any internet connection.

## Supported Data Formats

| File type | Content |
|---|---|
| `*.mbtiles` | Map tiles (vector or raster) |
| `*.etiles` | Elevation / hillshade data |
| `*.vtiles` | Offline routing (Valhalla format) |
| `nutigeodb.*` | Offline geocoding / reverse geocoding |

## Where to Put Data Files

### Android

Place your data packs in the **`alpimaps_mbtiles`** folder on your device storage (internal or SD card). The folder structure should look like this:

```
alpimaps_mbtiles/
├── world.mbtiles          ← World-level map (low zoom)
├── routes.mbtiles         ← World hiking/cycling routes
├── france/
│   ├── france_full.mbtiles   ← Full France OSM map
│   ├── france.etiles         ← Elevation / hillshade
│   ├── france_contours.mbtiles
│   ├── france_routes.mbtiles
│   ├── france.vtiles         ← Offline routing
│   └── nutigeodb.*           ← Offline geocoding
└── switzerland/
    └── ...
```

You can change the base data folder in **Settings → Offline Data → Map Data Path**.

### iOS

On iOS, place files in the **AlpiMaps Documents** folder (accessible via the Files app or iTunes File Sharing). The same folder structure applies.

## What Each File Enables

| File | Unlocks |
|---|---|
| `*_full.mbtiles` | Full offline vector map for the region |
| `*.etiles` | Hillshade rendering, slope overlay, per-point elevation |
| `*_contours.mbtiles` | Elevation contour lines on the map |
| `*_routes.mbtiles` | Hiking/cycling route overlay at all zoom levels |
| `*.vtiles` | Offline turn-by-turn routing with Valhalla |
| `nutigeodb.*` | Offline geocoding (search) and reverse geocoding |

## Automatic Detection

On app start AlpiMaps scans the data folder and automatically loads everything it finds. The map UI reflects which features are available — for example, the contour toggle only appears when contour data is present.

## Third-Party MBTiles Sources

You can use MBTiles files from other sources (e.g. downloaded from the web). Any valid MBTiles file containing vector or raster tiles can be added as a custom layer via the Layers panel (tap **+**).

Note: elevation-specific features (hillshade, slope, elevation profiles) require etiles files generated by the AlpiMaps data generator, which use a specific encoding.

## Tile Cache

Online tiles are cached locally so they remain available after first load. You can clear the cache for a specific layer from the Layer Options (tap ⋮ next to any layer in the Layers panel).

## Local Tile Server

AlpiMaps includes a built-in HTTP tile server that can serve your local MBTiles files to any other app or browser on the same device or local network.

- Enable it in **Settings → Behavior → Auto-start Tile Server**
- Default port: **8081** (configurable in Settings → Behavior → Tile Server Port)

This is useful for loading tiles in third-party apps that accept XYZ tile URLs.

## Offline Data Path Settings

You can customise where AlpiMaps looks for data files:

- **Settings → Offline Data → Map Data Path** — base folder for MBTiles/etiles/vtiles/nutigeodb files
- **Settings → Offline Data → Items Data Path** — folder where saved markers and routes are stored

Both paths can be set to any accessible folder, including SD card locations on Android.
